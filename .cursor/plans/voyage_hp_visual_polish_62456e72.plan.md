# VOYAGE HP レイアウト脱テンプレ化

## 課題

CSSのスタイリングは十分だが、**レイアウトパターンが「AI生成テンプレ」のまま**。
「枠の中にアイコン + タイトル + 説明文」のカードグリッドが4箇所で繰り返されており、単調かつプロのLPらしさに欠ける。

## 改善対象と方針

全て [src/App.jsx](src/App.jsx) 内の変更。

---

### 1. お悩みセクション（PainPoints / L423-464）

**現状**: 5枚の吹き出しカード + 人物アイコンが `grid-cols-5` で均一に並ぶ
**問題**: カードが小さく情報が読みにくい。吹き出し + 丸アイコンがテンプレ的

**変更後**: カードを廃止し、**縦積みチェックリスト形式**にする

- 各項目はフルワイド行。左にタグバッジ（採用/集客）、中央にキーワード（太字）、右に補足テキスト
- 項目間はボーダーで区切る。カードの枠・吹き出し・人物アイコンは不要
- PainCard コンポーネントは削除し、PainPoints 内にインライン実装

```
┌────────────────────────────────────────────────────┐
│ [採用]   応募の質が上がらない        求人媒体や... │
│────────────────────────────────────────────────────│
│ [採用]   従業員が定着しない          企業文化へ... │
│────────────────────────────────────────────────────│
│ [採用]   面談前に離脱される          せっかく...   │
│────────────────────────────────────────────────────│
│ [集客]   リピートに繋がらない        SNS広告や... │
│────────────────────────────────────────────────────│
│ [集客]   何を配信すれば...           LINE公式を... │
└────────────────────────────────────────────────────┘
```

### 2. リクエル特徴（Service内 / L779-793）

**現状**: 3枚の `bg-[#f7faf7]` カード。各カードに中央寄せアイコン + タイトル + 説明
**問題**: 最もよくあるAIテンプレの「3カラムカード」パターン

**変更後**: カードを廃止し、**3カラムのナンバリング形式**にする

- 各特徴は `01` / `02` / `03` のナンバーで始まる
- カード枠なし。区切りは左ボーダー `border-l-2 border-[#06C755]` のみ
- アイコン画像は削除。テキストの力で伝える

```
  01                        02                        03
  即時性                    自動化                    可視化
  ──                        ──                        ──
  開封率80%の到達力で       日程調整・Q&A対応を       スコアリングで候補者
  候補者との接点を維持      システム化し工数圧縮      の志望度を定量化
```

### 3. L-ACADEMIA サービス一覧（Service内 / L806-816）

**現状**: 6枚の `bg-[#f7faf7]` カードが `grid-cols-3` で並ぶ。各カードにアイコン + タイトル + 説明
**問題**: 6枚同形カードの繰り返しが最もテンプレ感が強い

**変更後**: カードグリッドを廃止し、**コンパクトな2列テキストリスト**にする

- 1つの `bg-[#f7faf7]` ブロック内に、左右2列 x 3行
- 各項目はタイトル（太字）+ 説明（1行）のみ。タイトル前に小さな緑丸ドット
- アイコン画像は使わない（情報のノイズを減らす）

```
┌─────────────────────────────────────────────────┐
│  ● 公式LINE開設・運用       ● 企画・シナリオ設計  │
│    目的に合わせた最適な...     ユーザー心理に...   │
│                                                   │
│  ● Lステップ構築・運用       ● WEBデザイン        │
│    高度なシナリオ自動化で...   リッチメニューや... │
│                                                   │
│  ● コピーライティング        ● アカウント分析     │
│    行動を促す文章設計で...     数値に基づく定期... │
└─────────────────────────────────────────────────┘
```

---

### 4. BACKSTAGE 採用支援/集客支援（BackstageGroup内 / L686-700）

**現状**: `grid-cols-2` で2枚のカード。各カードにアイコン + ラベル + キーワード + 説明
**問題**: 小さなカード2枚並びが窮屈でテンプレ的

**変更後**: カードを廃止し、**全幅2カラム対比レイアウト**にする

- 親の `bg-[#f7faf7]` ブロックはそのまま
- 中央に細い縦ディバイダー（PCのみ）
- 左半分: 採用支援。キーワードを大きなテキストで表示
- 右半分: 集客・売上支援。同様にキーワードを大きく
- アイコンはラベルの横にごく小さくインライン配置
- モバイルでは縦積み、間にグリーンの水平線を配置

```
  採用支援                      │  集客・売上支援
                                │
  採用コストを削減し、          │  売上に直結する
  定着率を改善                  │  リスト資産を構築
                                │
  LINEのシナリオ設計で          │  「フロー」を「ストック」へ。
  「志望度」へと変換...         │  ユーザーをLINEで見込み客...
```
